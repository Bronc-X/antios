## 1. 使命与边界

### 1.1 项目使命
本项目旨在构建一个“反焦虑闭环”iOS 应用：
- Max 主动问询
- 每日校准
- 科学解释
- 行动跟进

目标是让用户在高压力状态下，快速获得可执行、可持续、可追踪的恢复路径。

### 1.2 目标用户
- 高压工作人群（20-45 岁）
- 焦虑恢复期用户
- 希望通过 Apple Watch / HealthKit 做数据化管理的用户

### 1.3 范围宪制
本期必须做：登录/会话恢复、临床基线、引导流程、Dashboard、Report、Max、Plans、Settings、多语言。

信息架构允许重构为更小的主壳层，只要上述能力仍然可达。
当前批准的重构方向：
- 主壳层可合并为 `Home / Coach / Me`
- 本地数据层可采用 SwiftData local-first
- 远端 Auth 与业务同步仍可继续使用现有 Supabase / App API

本期明确不做：
- 社区社交
- 支付订阅闭环
- Android/Web 同步发布
- 非 Apple 可穿戴接入

## 3. 工程红线

### 3.1 安全红线
- 禁止提交真实密钥、token、证书。
- 禁止在日志、截图、报告里泄露敏感信息。
- 若泄露：立即吊销 -> 轮换 -> 追溯影响 -> 记录复盘。

### 3.2 变更红线
- 禁止无批准新增核心依赖。
- 禁止跳过验证直接合并“看起来可用”的代码。
- 禁止用硬编码临时妥协替代系统化修复。

### 3.3 破坏性操作红线
- 未获明确授权，不得执行破坏性删除或不可逆重写。

## 4. 质量门禁（Definition of Done）

单项任务完成必须同时满足：
- 实现与当前计划步骤一致（不越界）
- 核心路径可运行
- loading/error/empty 状态可用
- 相关治理记录已同步到 C/D 文件
- 触发语音通知：`say "master job done"`

## 5. 协作规则（人类 + AI）

- AI 是执行器，不是需求决策者。

- 每次纠错必须沉淀经验，禁止重复同类错误。
